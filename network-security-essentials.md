# Network Security Essentials

Learn about key aspects of network security essentials and how to monitor and protect against adversaries. 

## Introduction

### Importance of Networks

- Networks form the backbone of modern organizations.
- Key components such as:
  - Servers
  - Workstations
  - Applications
  - Security devices  
  are interconnected into a single ecosystem.

### Network Perimeter

- The network perimeter separates the internal environment from the external Internet.
- It is a primary target for attackers.
- Acts as the first line of exposure to potential threats.

## Network Overview

### Overview

- A computer network is an organized system of interconnected assets enabling:
  - Communication
  - Resource sharing
  - External connectivity
- Understanding each component helps identify suspicious activity from a security perspective.

---

### Key Components in a Small Enterprise Network

#### 1. User Workstations (Endpoints)

- Devices: PCs, laptops used by employees
- Common attack entry points via:
  - <phishing> emails
  - Malicious downloads  

**Security Importance**

- Often less monitored but critical for initial compromise
- Provide attackers a foothold for lateral movement

**Monitoring Insight**

- Endpoint logs show malicious processes
- Network logs may reveal <C2> (Command & Control) connections early

---

#### 2. File & Database Servers

- Store critical business data:
  - File servers → shared documents
  - Database servers → structured data (HR, finance, customers)

**Security Importance**

- High-value targets for:
  - Data theft (exfiltration)
  - Ransomware attacks
- Compromise leads to major business impact

---

#### 3. Application Servers (Web, Email, <VPN>, etc.)

- Provide essential services:
  - Web servers → websites/apps
  - Email servers → communication
  - Gateways (<VPN>) → remote access

**Security Importance**

- Externally exposed → frequently targeted
- Vulnerable to misconfigurations and exploits

**What to Monitor**

- Application logs, <IDS/IPS> alerts, and <attack> signatures

**Detectable Threats**

- Exploits (e.g., <SQL> injection)
- Brute-force attacks on email or <VPN> services
- Suspicious external <IP addresses> interacting with apps

---

#### 4. Active Directory (<AD>) / Authentication Servers

- Central identity management system
- Manages:
  - Users, groups, devices
  - Access permissions
- Users authenticate using <domain> credentials

**Security Importance**

- <AD> controls access across the network
- Prime target for attackers:
  - Privilege escalation
  - <credential dumping>
  - Lateral movement

**Monitoring Focus**

- Authentication logs for:
  - Failed logins (password spraying)
  - Logins from unusual <locations>
  - Abnormal access behavior

---

#### 5. Routers & Switches (Network Infrastructure)

- Routers → connect networks (LAN ↔ Internet)
- Switches → connect internal devices

**Security Importance**

- If compromised, attackers can:
  - Intercept traffic (MITM attacks)
  - Reroute traffic
  - Create hidden communication channels

---

#### 6. Firewalls / Perimeter Devices

- A <firewall> controls traffic between internal and external networks
- Functions:
  - Packet filtering
  - Deep inspection
  - Intrusion prevention
  - Malware detection

**Security Importance**

- First line of defense
- Prevents unauthorized access to services (e.g., databases or <application servers>)
- Provides critical logs

**Monitoring Value**

- Firewall logs reveal:
  - Port scans
  - Brute-force attempts
  - Exploitation attempts

## Network Visiblity

### Overview

- Network visibility = ability to monitor and understand network activity
- Core principle: *“You can’t defend what you can’t see”*
- Enables:
  - Threat detection
  - Incident investigation
  - Strong security posture
- Focus is on building a clear picture, not capturing every packet
- Without visibility → <no awareness> of threats

---

### Why Visibility is Crucial

- Acts as the “eyes” of a network (like cameras in a house)
- Without it, attacks may go unnoticed:
  - Malware infections
  - Unauthorized access
  - Data exfiltration

**Key Capabilities Enabled**

- Detect anomalies
- Investigate incidents
- Perform threat hunting
- Ensure compliance

---

### Log Sources for Visibility

- Visibility comes primarily from **logs**
- Two main categories:
  1. Host-Centric Logs
  2. Network-Centric Logs

---

#### 1. Host-Centric Logs

##### Overview

- Generated by individual devices:
  - Servers
  - Workstations
  - Laptops
- Provide detailed, system-level visibility

##### Key Sources

- **Operating System Logs**
  - Windows Event Logs
  - <Linux> syslog
  - macOS logs  

- **Application Logs**
  - Web servers (<Apache>, Nginx)
  - Databases (MySQL, MSSQL)

- **Security Tool Logs**
  - Antivirus
  - Endpoint Detection & Response (<EDR>)
  - Host-based intrusion detection systems (<HIDS>)

##### Importance

- Enables deep forensic analysis:
  - Tracks file access/modifications
  - Detects malicious processes
  - Monitors user activity
- Identifies:
  - Script execution (e.g., <PowerShell>)
  - System changes (registry, <file system>)
- Helps assess malware impact

---

#### 2. Network-Centric Logs

##### Overview

- Generated by network devices monitoring traffic
- Show interactions **between systems**

##### What They Capture

- Source & destination <IP addresses>
- Ports and protocols
- Traffic actions (allowed/blocked)

##### Key Sources

- **Firewalls**
  - <firewall> logs track allowed/denied traffic
  - First indicator of external threats

- **Intrusion Detection/Prevention Systems**
  - (<IDS>/<IPS>)
  - Detect known attack signatures and anomalies

- **Routers & Switches**
  - Provide flow data (who talked to whom, duration, volume)

- **Web Proxies**
  - Track user web activity
  - Detect web-based threats and policy violations

- **<VPN> Systems**
  - Monitor remote access:
    - Who connected
    - From where
    - When

##### Importance

- Provides a “bird’s-eye view” of the network

**Key Security Benefits**
- Early threat detection (e.g., malicious <IP addresses>)
- Detect Command & Control (<C2>) activity
- Track lateral movement
- Identify data exfiltration
- Understand attack scope

## Network Perimeter

### Overview

- The **network perimeter** is the boundary between:
  - Internal network (<trusted zone>)
  - External Internet (<untrusted zone>)
- Acts as the main entry/exit point for all traffic
- Considered the **first and most critical line of defense**

**Key Concepts**

- Internal network → business-critical systems
- External Internet → potential threats
- Perimeter → controlled gateway for all traffic

---

### What Defines the Perimeter

- Not just physical devices — includes:
  - Hardware
  - Virtual gateways
  - Cloud connections
  - Remote access points

---

### Common Perimeter Components

- **Firewalls**
  - Filter and control traffic

- **Routers / Gateways**
  - Route traffic and enforce rules

- **Demilitarized Zone (<DMZ>)**
  - Isolated network for public-facing services:
    - Web servers
    - Mail servers
    - <DNS> servers

- **Remote Access Gateways / <VPN>**
  - Secure access for remote users

---

### Importance of the Network Perimeter

- Acts as a **gatekeeper** controlling inbound/outbound traffic
- Made up of multiple security controls working together
- First place where:
  - Attacks are observed
  - Defenses are applied

---

### Common Attack Vectors at the Perimeter

If misconfigured or weak, attackers can:

- Exploit exposed services:
  - <SSH>, MySQL, <RDP>
- Perform reconnaissance and scanning
- Launch brute-force attacks
- Exfiltrate sensitive data

---

### Example: Small Enterprise Perimeter

**Typical Setup**

- A <firewall> sits between Internet and internal LAN
- Public web server placed in a <DMZ>
- Internal systems (<email>, file, database) protected behind the firewall
- Remote employees connect via <VPN> gateway

**Result**

- Controlled access to internal systems
- Public services isolated for safety

---

### Why the Perimeter Matters

- First line of defense against external threats
- Attackers scan perimeter <systems> for:
  - Open ports
  - Vulnerabilities

**Risks of Weak Perimeter**

- Unauthorized access (e.g., exposed <RDP>/<SSH>)
- Data breaches
- Malware infections

---

## Role of a Security Analyst

**Key Responsibilities**

- Monitor <firewall> logs:
  - Allowed vs blocked traffic
- Detect:
  - Scanning activity
  - Brute-force attempts
- Identify unusual outbound traffic:
  - Malware beaconing
  - Data exfiltration
- Ensure only necessary services are exposed

## Network Perimeter Monitoring and Protecting

### Overview

- Attackers constantly probe the network perimeter for weaknesses
- Monitoring involves:
  - Firewalls
  - Intrusion Detection/Prevention Systems (<IDS>/<IPS>)
  - Access control mechanisms
- Goal: detect and respond to threats early

---

### Why Monitor the Perimeter

Enables security analysts to:

- Detect early-stage attacks (port scans, brute-force)
- Identify misconfigurations exposing services
- Spot abnormal outbound traffic (malware, data exfiltration)

---

### Monitoring Scenarios

#### Scenario 1: Port Scanning (Reconnaissance)

- An attacker probes multiple ports on a target system

##### Firewall Log  

```text
2025-09-22 08:30:04 ALLOW TCP 198.51.100.45:49876 -> 10.0.0.51:80
2025-09-22 08:30:05 BLOCK TCP 203.0.113.10:50001 -> 10.0.0.20:21
2025-09-22 08:30:06 BLOCK TCP 203.0.113.10:50002 -> 10.0.0.20:22
2025-09-22 08:30:07 ALLOW TCP 192.0.2.115:51235 -> 10.0.0.50:443
2025-09-22 08:30:08 BLOCK TCP 203.0.113.10:50003 -> 10.0.0.20:23
2025-09-22 08:30:09 BLOCK TCP 203.0.113.10:50004 -> 10.0.0.20:25
2025-09-22 08:30:10 ALLOW TCP 198.51.100.92:51111 -> 10.0.0.50:443
2025-09-22 08:30:11 BLOCK TCP 203.0.113.10:50005 -> 10.0.0.20:53
```

##### Log Breakdown

    The same external IP (203.0.113.10) is trying to connect to multiple ports on the same internal machine quickly.
    Analyst's Verdict: This is a classic port scan. The attacker is looking for an open service to target.

#### Scenario 2: Web Attacks (SQL Injection, XSS, etc.)

The company website is being attacked. An intrusion detection system (IDS) provides more details than a firewall, identifying the type of attack.

##### Web Application Firewall (WAF) Log

```text
timestamp=2025-09-22T09:14:44Z src_ip=192.0.2.130 action=ALLOW request="GET /index.html"
timestamp=2025-09-22T09:14:45Z src_ip=198.51.100.92 action=ALLOW request="GET /products.php?id=9"
timestamp=2025-09-22T09:14:46Z src_ip=[REDACTED] action=BLOCK request="GET /search.php?q=<script>alert('XSS')</script>" rule_id=941100 attack_type="XSS"
timestamp=2025-09-22T09:14:47Z src_ip=192.0.2.140 action=ALLOW request="GET /css/style.css"
timestamp=2025-09-22T09:15:42Z src_ip=[REDACTED] action=BLOCK request="GET /../../../../etc/passwd" rule_id=930120 attack_type="Directory Traversal"
```

##### Log Breakdown

    This log shows a mix of ALLOW and BLOCK actions. An analyst can filter for action=BLOCK to immediately find threats. The WAF does the hard work by blocking the request and telling us why.
    The attack_type="SPL Injection" alert shows the attacker trying to dump database information.
    The attack_type="XSS" alert is an attempt to inject malicious scripts.
    The attack_type="Directory Traversal" alert shows an attempt to read sensitive server files.
    Analyst's Verdict: The WAF successfully identifies and blocks multiple web attack types from suspicious IP. This is a high-confidence alert that an attacker is actively targeting the website.

---

#### Scenario 3: VPN Brute-Force Attack

An attacker is trying to guess a user's password to gain remote access to the network. This creates a lot of noise in the authentication logs.  

##### VPN Gateway Log

```markdown
2025-09-22 10:12:11 FAILED_AUTH TCP [REDACTED]:31245 -> 10.0.0.1:443 (user 'admin')
2025-09-22 10:12:15 FAILED_AUTH TCP [REDACTED]:31248 -> 10.0.0.1:443 (user 'admin')
2025-09-22 10:12:21 SUCCESS_AUTH TCP 198.51.100.88:41233 -> 10.0.0.1:443 (user 'b.jones')
2025-09-22 10:12:08 FAILED_AUTH TCP [REDACTED]:31249 -> 10.0.0.1:443 (user 'guest')
2025-09-22 10:12:09 FAILED_AUTH TCP [REDACTED]:31250 -> 10.0.0.1:443 (user 'user')
```

##### Log Breakdown

    The log is flooded with FAILED_AUTH and SUCCESS_AUTH events.
    A few successful logins from various IPs are normal.
    The problem is the massive volume of failures.
    To find the attack, an analyst would filter or group the logs by source IP.
    Doing so would immediately reveal that a suspicious user is responsible for hundreds of failed login attempts in a very short period.
    Analyst's Verdict: One suspicious IP is observed conducting a brute-force attack against the VPN gateway. The attacker is attempting to use a list of common usernames ('admin', 'root', 'test', etc.) to find a valid account to compromise. The scattered successful logins are benign traffic from legitimate employees.

##### Key Takeaways

    An analyst's job is to separate normal traffic from suspicious activity.
    Look for suspicious patterns.
    Repetition from one source to many destinations = Scanning.
    Repetition from one source to one destination = Brute-forcing.
    Traffic at perfect, regular intervals = Malware beaconing.
    Context is key. An IDS alert telling you why something was flagged is more valuable than a simple firewall block.
    Monitoring the network perimeter is the first step in detecting attacks
